# 클로드 코드, 더 깊이 들여다보기

> 책에서 미처 다루지 못한 클로드 코드의 숨은 기능들을 한 번에 정리합니다.

## 들어가며

『바로 배워 바로 쓰는 클로드』를 집필하면서, 원고 분량의 한계로 아쉽게 빠진 내용들이 있었습니다. 이 부록에서는 클로드 코드의 세세한 기능들을 한꺼번에 정리해 둡니다. 처음부터 끝까지 정독하기보다 필요할 때 해당 섹션을 찾아보는 레퍼런스로 활용하면 좋습니다.

## 1. 슬래시 커맨드

입력창에서 `/`를 누르면 즉시 명령 팔레트가 열립니다. 자주 쓰이는 기본 커맨드부터 정리합니다.

### 세션과 UI

| 커맨드 | 설명 |
|--------|------|
| `/help` | 전체 커맨드 목록과 도움말 |
| `/clear` | 대화 내용과 컨텍스트를 모두 비움 |
| `/compact` | 대화를 요약해 컨텍스트를 압축 |
| `/cost` | 현재 세션의 토큰/비용 사용량 |
| `/model` | 사용할 모델 선택 (Opus/Sonnet/Haiku) |
| `/config` | 설정 화면 열기 |
| `/exit` | 세션 종료 |

### 프로젝트와 작업

| 커맨드 | 설명 |
|--------|------|
| `/init` | 프로젝트에 맞는 `CLAUDE.md`를 자동 생성 |
| `/memory` | 메모리(CLAUDE.md) 파일들을 편집 |
| `/review` | 현재 PR/변경 사항 리뷰 |
| `/pr_comments` | 깃허브 PR의 코멘트를 가져와 반영 |
| `/bug` | 버그 리포트 작성 도우미 |

### 커스텀 슬래시 커맨드 (스킬)

`~/.claude/commands/` 또는 프로젝트의 `.claude/commands/` 폴더에 `.md` 파일을 만들면 그 자체가 슬래시 커맨드가 됩니다. 파일 이름이 커맨드 이름이고, 내용이 프롬프트입니다.

```markdown
---
description: 현재 브랜치의 변경 사항을 한국어로 리뷰
---

변경된 파일들을 읽고, 다음 기준으로 리뷰를 작성해 주세요:

1. 버그 가능성
2. 성능 이슈
3. 네이밍과 가독성

모든 설명은 한국어로 해주세요.
```

이 파일을 `.claude/commands/ko-review.md`로 저장하면 `/ko-review`로 실행됩니다. 팀 전체가 쓰는 커맨드라면 프로젝트 폴더에, 개인용이면 홈 디렉터리에 두면 됩니다.

## 2. CLAUDE.md와 메모리 시스템

클로드 코드는 세션 시작 시 여러 위치의 `CLAUDE.md` 파일을 자동으로 읽어 맥락으로 사용합니다.

### 계층 구조

1. **엔터프라이즈 수준** — 조직 전체에 적용되는 정책
2. **사용자 수준** — `~/.claude/CLAUDE.md`, 모든 프로젝트에 공통
3. **프로젝트 수준** — 프로젝트 루트의 `CLAUDE.md`, 팀원 모두와 공유 (git 커밋 대상)
4. **하위 디렉터리** — 특정 폴더에만 적용되는 지침

아래 계층의 지침이 위 계층의 지침과 충돌하면, 더 구체적인(아래) 지침이 우선합니다.

### 자주 쓰는 패턴

```markdown
# 프로젝트 개요

이 저장소는 Next.js 기반 대시보드입니다. 주요 디렉터리:
- `app/` — 라우트와 페이지
- `components/ui/` — shadcn 컴포넌트 (수정 금지, 새 컴포넌트로 확장)
- `lib/api/` — 백엔드 클라이언트

# 개발 규칙

- 커밋 메시지는 한국어로 작성합니다.
- 테스트는 `bun test`로 실행합니다.
- `any` 타입은 쓰지 않습니다.

# 참조

@docs/architecture.md
@.cursorrules
```

`@파일경로` 문법으로 다른 문서를 임포트할 수 있습니다. 아키텍처 문서, 코딩 컨벤션 등을 CLAUDE.md에서 불러오면 관리가 쉬워집니다.

### `#`으로 메모리 빠르게 추가

입력창 맨 앞에 `#`을 붙이고 문장을 쓰면, 그 내용을 CLAUDE.md의 어느 파일에 저장할지 묻는 메뉴가 나옵니다. 대화 중 발견한 규칙을 즉시 기록할 때 유용합니다.

## 3. 훅(Hooks)

훅은 클로드 코드의 특정 이벤트에 맞춰 자동 실행되는 셸 명령어입니다. `~/.claude/settings.json` 또는 프로젝트의 `.claude/settings.json`에 정의합니다.

### 주요 훅 이벤트

| 이벤트 | 언제 실행되나 |
|--------|---------------|
| `PreToolUse` | 도구 호출 직전 — 검증, 차단 가능 |
| `PostToolUse` | 도구 호출 완료 후 — 결과 후처리 |
| `UserPromptSubmit` | 사용자가 메시지를 보낸 직후 |
| `Stop` | 응답이 끝났을 때 |
| `Notification` | 알림 발생 시 |
| `SessionStart` / `SessionEnd` | 세션 시작/종료 |
| `PreCompact` | 자동 압축 직전 |
| `SubagentStop` | 서브에이전트가 끝났을 때 |

### 실전 예시: 저장 시 자동 포맷팅

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "bun run format:fix"
          }
        ]
      }
    ]
  }
}
```

파일을 쓰거나 편집할 때마다 프로젝트 포맷터가 실행됩니다. ESLint 자동 수정, 프리티어 적용, 타입 체크 등을 훅으로 엮으면 "클로드가 뭘 했든 결과는 일관된 스타일"을 보장할 수 있습니다.

### 위험한 명령 차단

`PreToolUse` 훅에서 특정 패턴을 발견하면 실행을 막을 수 있습니다. `rm -rf /` 같은 위험한 Bash 호출을 사전 검증하는 용도로 자주 씁니다.

## 4. MCP 서버

MCP(Model Context Protocol)는 클로드가 외부 도구와 소통하는 표준 프로토콜입니다. 책에서는 기본 개념만 다뤘지만 실제 활용 범위는 훨씬 넓습니다.

### 전송 방식

- **stdio** — 로컬 프로세스와 표준 입출력으로 통신 (가장 흔함)
- **SSE / HTTP** — 원격 MCP 서버와 HTTP로 통신

### 파일시스템 MCP

가장 기본적인 MCP 서버로, 특정 경로에 읽기/쓰기 접근을 허용합니다.

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/me/docs"]
    }
  }
}
```

### 자주 쓰는 서버들

| 서버 | 용도 |
|------|------|
| `server-filesystem` | 지정 경로 파일 접근 |
| `server-github` | 이슈, PR, 저장소 조작 |
| `server-postgres` / `server-sqlite` | SQL 질의 실행 |
| `server-slack` | 슬랙 메시지 읽기/쓰기 |
| `figma-mcp` (공식) | 피그마 디자인 → 코드 |
| `playwright-mcp` | 브라우저 자동화 |

### 프로젝트 단위 MCP 설정

프로젝트 루트에 `.mcp.json`을 두면 팀원 모두가 같은 MCP 서버를 공유합니다. 민감한 서버는 개인 설정에만 두고, 팀 공용은 `.mcp.json`에 두는 분리가 좋습니다.

## 5. 서브에이전트

서브에이전트는 메인 대화와 독립된 컨텍스트에서 작업을 수행하는 별도의 클로드 인스턴스입니다. 큰 이점은 두 가지입니다.

1. **컨텍스트 보호** — 긴 조사 결과가 메인 세션 히스토리를 오염시키지 않음
2. **병렬 실행** — 여러 에이전트를 동시에 띄울 수 있음

### 빌트인 에이전트

- `general-purpose` — 범용 조사·탐색
- `Explore` — 빠른 코드베이스 탐색
- `Plan` — 구현 계획 수립
- `claude-code-guide` — 클로드 코드 자체 사용법 질문

### 커스텀 에이전트 만들기

`.claude/agents/` 또는 `~/.claude/agents/`에 마크다운 파일을 만듭니다.

```markdown
---
name: korean-reviewer
description: 한국어로 코드 리뷰를 수행합니다
tools: Read, Grep, Glob
model: sonnet
---

당신은 시니어 한국어 코드 리뷰어입니다.
버그, 성능, 가독성 순으로 지적하세요.
```

메인 클로드가 Agent 도구로 이 에이전트를 호출할 수 있고, `/agents` 같은 커맨드로 직접 관리할 수도 있습니다.

## 6. 퍼미션과 설정 파일

### 설정 파일 계층

| 파일 | 위치 | 용도 |
|------|------|------|
| 엔터프라이즈 | OS별 시스템 경로 | 조직 전체 정책 (읽기 전용) |
| 사용자 설정 | `~/.claude/settings.json` | 개인 기본값 |
| 프로젝트 설정 | `.claude/settings.json` | 팀 공유 (git 커밋) |
| 프로젝트 로컬 | `.claude/settings.local.json` | 개인용 (gitignore) |

아래로 갈수록 우선순위가 높습니다.

### 퍼미션 규칙

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run test)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "Read(~/projects/**)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Write(.env*)"
    ],
    "ask": [
      "Bash(git push:*)"
    ]
  }
}
```

- `allow` — 승인 없이 바로 실행
- `deny` — 절대 실행 금지
- `ask` — 매번 사용자에게 물음 (기본값)

반복적으로 승인하게 되는 명령은 `/less-permission-prompts` 같은 스킬로 자동 수집해 `allow`에 추가할 수 있습니다.

## 7. IDE 통합

### 지원 환경

- **VS Code** / **Cursor** / **Windsurf** — 공식 확장
- **JetBrains** (IntelliJ, WebStorm, PyCharm 등) — 공식 플러그인
- **Vim / Neovim** — `claude.vim` 등 커뮤니티 플러그인

IDE에서 실행하면 현재 열린 파일, 선택 영역, diagnostics(경고/에러)가 자동으로 클로드에 전달됩니다.

### `/ide` 커맨드

터미널 세션에서 `/ide`를 입력하면 현재 실행 중인 IDE에 세션을 연결할 수 있습니다. 선택 영역을 자동으로 참조할 수 있고, 변경 사항이 IDE의 diff 뷰에 표시됩니다.

## 8. 모드 전환

### 플랜 모드(Plan Mode)

파일을 쓰지 않고 조사와 계획만 수행하는 모드입니다. 대규모 작업을 시작하기 전에 "계획을 먼저 세우고 승인받는" 워크플로를 만들 수 있습니다.

- 진입: `Shift+Tab`을 눌러 모드 순환, 또는 `/plan`
- 종료: 계획이 마음에 들면 승인해 실제 구현으로 전환

### 자동 수락 모드(Auto-accept)

도구 호출을 매번 승인하지 않고 자동으로 진행하는 모드입니다. 반복 작업이 많을 때 편리하지만, **신뢰할 수 있는 환경에서만** 사용해야 합니다.

### 모드 순환

`Shift+Tab`을 반복해 누르면 기본 → 자동 수락 → 플랜 → 기본 순서로 순환합니다.

## 9. 입력창 단축 문법

| 기호 | 기능 |
|------|------|
| `/` | 슬래시 커맨드 |
| `@` | 파일/폴더 참조 (자동완성) |
| `#` | CLAUDE.md에 메모 추가 |
| `!` | Bash 모드 — 해당 명령을 셸에서 실행 |

`!` 모드는 대화를 끊지 않고 `ls`, `git status` 같은 명령을 확인하고 싶을 때 특히 유용합니다.

## 10. 키보드 단축키

| 키 | 동작 |
|-----|------|
| `Esc` | 현재 도구 호출/응답 중단 |
| `Esc` 두 번 | 이전 메시지로 되돌아가 편집 (체크포인트) |
| `Ctrl+C` | 현재 입력 취소 / 세션 종료 확인 |
| `Ctrl+D` | 세션 종료 |
| `Ctrl+R` | 입력 히스토리 역검색 |
| `Ctrl+V` | 클립보드의 이미지 붙여넣기 |
| `Shift+Tab` | 모드 순환 (기본 / 자동 수락 / 플랜) |
| `Tab` | 자동완성 |

`Esc` 두 번의 되돌리기(체크포인트)는 실수로 잘못된 경로를 갔을 때 가장 먼저 떠올려야 할 기능입니다.

## 11. CLI 플래그와 헤드리스 모드

### 기본 플래그

| 플래그 | 용도 |
|--------|------|
| `-p "프롬프트"` | 헤드리스 모드 — 한 번 실행하고 종료 |
| `--continue` | 가장 최근 세션 이어서 진행 |
| `--resume` | 세션 목록에서 선택해 재개 |
| `--model opus` | 특정 모델 지정 |
| `--add-dir <path>` | 작업 디렉터리 추가 |
| `--mcp-config <file>` | MCP 설정 파일 지정 |
| `--output-format json` | 파이프라인에서 쓰기 쉬운 출력 |

### 헤드리스 활용 예

CI/CD에서 PR 리뷰를 자동화하거나, 스크립트로 대량 작업을 돌릴 때 씁니다.

```bash
claude -p "docs/ 아래 모든 마크다운의 오탈자를 수정해 줘" \
  --output-format stream-json | jq .
```

셸 파이프와 결합하면 클로드 코드는 "대화형 도구"를 넘어 "스크립트 가능한 AI 빌드 블록"이 됩니다.

## 12. 세션 관리와 재개

- **자동 저장** — 모든 세션은 자동으로 저장됨
- **`--continue`** — 마지막 세션 이어서
- **`--resume`** — 여러 세션 중 선택
- **`/compact`** — 긴 대화를 요약으로 압축해 컨텍스트 확보
- **`/clear`** — 맥락을 완전히 비우고 처음부터 시작

컨텍스트 창이 한계에 가까워지면 자동 압축이 동작하지만, 중요한 구분점(한 작업이 끝나고 다른 작업을 시작할 때 등)에서는 수동으로 `/clear`하는 것이 깔끔합니다.

## 13. 파일과 이미지 입력

### 파일 참조

입력창에서 `@`를 누르면 현재 디렉터리의 파일/폴더가 자동완성됩니다. `@src/api/users.ts`처럼 참조하면 해당 파일을 클로드가 읽도록 지시합니다.

### 이미지 입력

- `Ctrl+V`로 클립보드 이미지를 붙여넣기
- 파일 경로를 드래그 앤 드롭
- 스크린샷 경로를 `@`로 참조

디자인 시안, 에러 스크린샷, 다이어그램을 그대로 보여주면 훨씬 정확한 답을 받을 수 있습니다.

## 14. 사고(Thinking) 모드

프롬프트에 특정 키워드를 포함하면 클로드가 더 긴 내부 사고를 수행합니다.

| 키워드 | 사고 깊이 |
|--------|-----------|
| `think` | 기본보다 길게 |
| `think hard` | 더 길게 |
| `think harder` | 훨씬 더 길게 |
| `ultrathink` | 최대 깊이 |

복잡한 아키텍처 결정, 까다로운 버그 분석에는 `ultrathink`를 고려할 만합니다. 대신 토큰 소비가 늘어납니다.

## 15. 모델 선택

`/model`로 언제든 바꿀 수 있습니다.

- **Opus** — 가장 강력, 복잡한 추론/설계에 적합
- **Sonnet** — 균형, 대부분의 코딩 작업에 적절
- **Haiku** — 빠르고 저렴, 단순 반복 작업에 적합

긴 세션에서는 작업 성격에 따라 모델을 바꿔가며 쓰는 것이 효율적입니다. 설계는 Opus로, 구현은 Sonnet으로, 단순 파일 이동/리네이밍은 Haiku로 같은 식입니다.

## 16. 환경 변수

클로드 코드의 동작을 환경 변수로 세밀하게 제어할 수 있습니다.

| 변수 | 용도 |
|------|------|
| `ANTHROPIC_API_KEY` | API 키 (로그인 대신 사용할 때) |
| `CLAUDE_CODE_USE_BEDROCK` | AWS Bedrock 경유 |
| `CLAUDE_CODE_USE_VERTEX` | Google Vertex AI 경유 |
| `ANTHROPIC_MODEL` | 기본 모델 오버라이드 |
| `DISABLE_TELEMETRY` | 텔레메트리 끄기 |
| `MAX_THINKING_TOKENS` | 사고 토큰 상한 |

기업 환경에서는 Bedrock/Vertex 경유 설정이 감사와 보안 요구를 맞추는 열쇠입니다.

## 17. 커스텀 상태줄

터미널 하단의 상태줄을 직접 만들 수 있습니다. 브랜치명, 모델명, 토큰 사용량 등을 실시간으로 표시하는 용도입니다.

```json
{
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh"
  }
}
```

스크립트는 JSON으로 세션 정보를 받아 한 줄 문자열을 출력하면 됩니다.

## 18. 비용과 사용량 추적

- `/cost` — 현재 세션의 토큰/달러 사용량
- 공식 대시보드 — API 계정에 로그인해 월간 집계 확인
- 헤드리스 출력의 `usage` 필드 — 스크립트에서 집계

`/compact`와 `/clear`를 적절히 섞으면 토큰 사용량을 체감할 수 있을 만큼 줄일 수 있습니다.

## 19. 플러그인과 스킬

### 스킬(Skill)

스킬은 특정 상황에서 자동으로 로드되는 지침 묶음입니다. `description` 필드에 트리거 조건을 적어두면 클로드가 해당 상황에서 스킬을 스스로 불러옵니다. 슬래시 커맨드와 달리 명시적으로 호출하지 않아도 됩니다.

### 플러그인 마켓플레이스

공식/커뮤니티에서 배포하는 스킬, 에이전트, 훅 세트를 한 번에 설치할 수 있는 방식입니다. 팀 단위로 워크플로를 표준화할 때 가장 편리한 선택입니다.

## 20. Git·깃허브 통합

- 커밋 메시지 자동 작성
- PR 생성 및 본문 작성 (`gh pr create`)
- PR 리뷰, 리뷰 코멘트 반영
- 브랜치 전략, 리베이스, 충돌 해결 도움

`gh` CLI가 설치되어 있으면 대부분의 깃허브 작업을 대화로 처리할 수 있습니다. 중요한 변경(force push, 브랜치 삭제)은 기본적으로 사용자에게 확인을 받으므로 안전합니다.

문서 링크를 주면 그 내용을 읽고 답하거나, 최신 라이브러리 변경 사항을 확인해 반영할 수 있습니다.

## 22. 백그라운드 작업과 워크트리

### 백그라운드 Bash

`run_in_background`로 시작한 명령은 개발 서버나 감시 프로세스처럼 오래 도는 작업에 적합합니다. 완료 시점에 알림을 받거나, 나중에 출력을 읽어올 수 있습니다.

### 깃 워크트리

`EnterWorktree` 기능은 에이전트가 별도의 작업 트리(격리된 브랜치 체크아웃)에서 작업하도록 해줍니다. 위험한 리팩터링을 메인 작업 트리와 분리해 실험할 때 유용합니다.

## 23. 스케줄과 반복 실행

- **`/loop`** — 특정 프롬프트나 커맨드를 주기적으로 실행
- **`/schedule`** — 크론 스케줄로 원격 에이전트 실행

"매시간 PR 큐를 확인해" 같은 주기적 작업을 클로드에게 위임할 수 있습니다.

## 24. 보안과 감사

- 모든 세션 로그가 로컬에 저장됨
- 엔터프라이즈 설정으로 특정 기능을 강제 비활성 가능
- Bedrock/Vertex 경유로 데이터 거버넌스 충족
- 퍼미션 `deny` 리스트로 위험 명령 사전 차단

민감한 저장소에서 작업할 때는 `/doctor`로 현재 설정을 점검하는 습관을 들이면 좋습니다.

## 25. 에이전트 SDK로 직접 만들기

클로드 코드 자체가 "터미널용 에이전트"이지만, 같은 구성 요소를 써서 **내 도구에 맞는 에이전트**를 만들 수도 있습니다. Claude Agent SDK는 도구 호출, 메모리, 권한, MCP 연결, 훅을 그대로 노출합니다.

```typescript
import { Agent } from '@anthropic-ai/agent-sdk';

const agent = new Agent({
  model: 'claude-opus-4-7',
  systemPrompt: '당신은 DB 마이그레이션 전담 에이전트입니다.',
  tools: [/* ... */],
  mcpServers: [/* ... */],
});

await agent.run('users 테이블에 last_login 컬럼 추가');
```

CLI 밖의 환경(슬랙 봇, 웹 앱, CI 잡)에 같은 능력을 심을 수 있다는 뜻입니다.
