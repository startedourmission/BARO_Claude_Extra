# 클로드 코드 기능 지도

> 이 부록의 "설명" 카테고리는 클로드 코드의 주요 기능을 주제별로 쪼갠 문서들입니다. 필요할 때 찾아보는 레퍼런스로 쓰세요. 아래는 각 문서가 무엇을 다루는지 한 줄 정리입니다.

## 기본 조작

### 슬래시 커맨드
`/` 로 실행하는 모든 것. 빌트인 명령 전체 목록, 커스텀 스킬 만드는 법, 프론트매터 전 필드, `$ARGUMENTS`·`` !` `` 같은 고급 문법까지.

### 모드 전환과 키보드 단축키
입력창 단축 문법 네 개(`/`, `@`, `#`, `!`), 모드 세 개(Normal, Auto-accept, Plan), 키보드 단축키 전체 치트시트. 매일 쓰는 UX의 기본.

### 퍼미션과 설정 파일
`allow`·`deny`·`ask` 규칙, 계층적 settings.json, 훅·상태줄·환경변수까지 — 클로드 코드의 "행동 규칙"을 설정하는 모든 지점.

## 맥락 관리

### CLAUDE.md와 메모리 시스템
세션마다 반복하는 설명을 박제하는 법. 엔터프라이즈·개인·프로젝트 계층, `@import` 문법, `#` 빠른 저장, AGENTS.md 표준.

### CLI·헤드리스 모드와 세션 관리
`claude -p`로 스크립트에 박아 넣는 법, 세션 재개, `/compact` vs `/clear`, 자동 압축 동작. CI·자동화 필수 지식.

### 모델·사고 모드·비용
Opus/Sonnet/Haiku 언제 쓸지, `think`·`ultrathink` 트레이드오프, 프롬프트 캐싱으로 비용 90% 줄이기, Bedrock/Vertex 경유.

## 자동화와 확장

### 훅(Hooks) 완전 가이드
도구 호출에 결정론적 자동화를 끼워 넣는 장치. 자동 포맷팅·위험 명령 차단·세션 요약 등 레시피 10개.

### MCP 서버
클로드와 외부 시스템을 잇는 표준 프로토콜. 설치·전송 방식·자주 쓰는 서버 목록·직접 만드는 법.

### 서브에이전트 설계
메인 컨텍스트를 깨끗하게 유지하며 작업을 위임·병렬화. 빌트인 에이전트, 커스텀 만들기, `context: fork`, 설계 패턴 4가지.

### 플러그인·스킬·Agent SDK
세 개념의 차이, 언제 뭘 쓰나. 팀에 배포할 플러그인, 내 앱에 심는 Agent SDK.
