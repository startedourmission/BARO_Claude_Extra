> 책에서 미처 다루지 못한 클로드 코드의 숨은 기능들을 소개합니다.

## 들어가며

『바로 배워 바로 쓰는 클로드』를 집필하면서, 원고 분량의 한계로 아쉽게 빠진 내용들이 있었습니다. 이 부록에서는 그런 이야기들을 하나씩 풀어보려 합니다.

## MCP 서버 활용하기

MCP(Model Context Protocol)는 클로드가 외부 도구와 소통하는 표준 프로토콜입니다. 책에서는 기본적인 개념만 다뤘지만, 실제로 활용할 수 있는 범위는 훨씬 넓습니다.

### 파일 시스템 MCP

로컬 파일에 접근하는 가장 기본적인 MCP 서버입니다.

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic-ai/mcp-filesystem"]
    }
  }
}
```

### 데이터베이스 연동

PostgreSQL, SQLite 등의 데이터베이스에 직접 질의할 수 있습니다. 데이터 분석 작업에서 특히 유용합니다.

## 프롬프트 엔지니어링 심화

책 본문에서 다룬 기본 원칙 외에, 실무에서 자주 쓰이는 고급 기법을 정리합니다.

### 체이닝 기법

하나의 복잡한 요청을 여러 단계로 나누어 처리하면 품질이 크게 향상됩니다.

1. **분석 단계** — 문제를 구조화하고 핵심 요소를 파악
2. **계획 단계** — 접근 방법을 설계
3. **실행 단계** — 실제 결과물을 생성
4. **검증 단계** — 결과를 검토하고 개선

### 컨텍스트 윈도우 관리

긴 대화에서는 컨텍스트 윈도우를 효율적으로 사용하는 것이 중요합니다.

| 전략 | 설명 | 효과 |
|------|------|------|
| 요약 삽입 | 긴 대화 중간에 요약을 요청 | 맥락 유지 |
| 파일 분리 | 긴 코드를 별도 파일로 관리 | 토큰 절약 |
| 역할 재설정 | 새 작업 시작 시 역할을 다시 부여 | 정확도 향상 |
