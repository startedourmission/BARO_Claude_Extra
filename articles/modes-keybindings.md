# 모드 전환과 키보드 단축키

> 매일 쓰는 UX 층. 단축 문법 네 가지(`/`, `@`, `#`, `!`)와 모드 세 가지(Normal, Auto-accept, Plan)만 제대로 알아도 생산성이 두 배가 됩니다.

## 입력창의 네 가지 단축 문법

### `/` — 슬래시 커맨드

빌트인 명령이나 커스텀 스킬 실행. 타이핑 시작하면 자동완성 메뉴 뜸.

```
/help
/clear
/compact
/ko-review
```

### `@` — 파일/폴더 참조

현재 디렉터리의 파일이나 폴더를 컨텍스트에 포함.

```
@src/api/users.ts
@docs/
@README.md
```

타이핑 시작하면 퍼지 매칭으로 자동완성. 여러 파일을 한 메시지에 `@` 여러 번 쓸 수도 있습니다.

```
@src/auth/login.ts @tests/auth.test.ts 이 두 파일 놓고 리뷰해 줘
```

### `#` — 메모리에 즉시 저장

대화 중 규칙을 CLAUDE.md에 바로 박제.

```
# 이 프로젝트에서는 axios 대신 ky를 쓴다
```

저장 위치 메뉴가 뜸 (개인/프로젝트/하위 디렉터리). "한 번 말해 두면 다음부터 자동 적용"의 가장 빠른 경로.

### `!` — Bash 모드

대화를 끊지 않고 셸 명령을 바로 실행.

```
!git status
!npm test
!ls -la
```

결과가 대화에 표시되지만, 클로드가 그 내용을 자동으로 해석하지는 않습니다. "참고하셔요" 느낌으로 보여주는 용도. 한 번의 실행으로 현재 상태를 확인하고 싶을 때 깔끔.

## 키보드 단축키 전체 표

### 핵심

| 키 | 동작 |
|-----|------|
| `Esc` | 진행 중인 도구 호출·응답 중단 |
| `Esc` 두 번 | 직전 메시지로 되돌아가 편집 (체크포인트) |
| `Ctrl+C` | 입력 취소 / 두 번 누르면 세션 종료 확인 |
| `Ctrl+D` | 세션 즉시 종료 |
| `Tab` | 자동완성 (파일·커맨드) |

### 히스토리·편집

| 키 | 동작 |
|-----|------|
| `↑` / `↓` | 이전/다음 입력 탐색 |
| `Ctrl+R` | 입력 히스토리 역검색 |
| `Ctrl+A` / `Ctrl+E` | 줄 처음/끝으로 |
| `Ctrl+W` | 단어 하나 지우기 |
| `Ctrl+U` | 현재 줄 전체 지우기 |

### 입력

| 키 | 동작 |
|-----|------|
| `Ctrl+V` | 클립보드 이미지 붙여넣기 |
| `Shift+Enter` | 줄바꿈 (여러 줄 입력) |
| `Enter` | 전송 |

### 모드

| 키 | 동작 |
|-----|------|
| `Shift+Tab` | 모드 순환 (Normal → Auto-accept → Plan → Normal) |

### 기타

| 키 | 동작 |
|-----|------|
| `Ctrl+L` | 화면 클리어 (대화는 유지) |

단축키는 운영체제·터미널·Vim 모드 활성 여부에 따라 달라질 수 있으니 `/help`로 현재 바인딩 확인 가능.

## 체크포인트 — `Esc` 두 번

가장 덜 알려진 강력 기능. 대화 도중 "아 여기서 방향이 잘못됐네"를 느끼면 `Esc` 두 번.

1. 직전 몇 개 메시지가 숨겨지며 편집 가능 상태로 바뀜
2. 내가 보냈던 프롬프트를 수정
3. 다시 전송하면 그 시점부터 재개

잘못된 경로로 갔을 때 "되돌리기" 도구. **신중한 커밋처럼 체크포인트를 의식적으로 활용**하면 긴 세션의 품질이 크게 올라갑니다.

## 모드 세 가지

클로드 코드는 작업 성격에 따라 세 가지 모드를 지원합니다. `Shift+Tab`으로 순환.

### Normal 모드 (기본)

- 도구 호출마다 사용자 승인 (설정에 따름)
- 가장 안전
- 신규 작업·민감 작업에 적합

### Auto-accept 모드

- 모든 도구 호출을 자동 승인 (deny 규칙은 여전히 적용)
- 빠른 반복 작업에 적합
- **신뢰할 수 있는 환경에서만**

상태줄에 `[AUTO]` 표시. 의도치 않게 켜진 상태로 위험 명령을 던지면 그대로 실행됩니다. 주의.

### Plan 모드

- 조사·계획만. **파일을 쓰지 않음**
- 탐색 도구(Read, Grep, Glob)는 가능
- Write/Edit/Bash는 차단

큰 작업 시작 전 "계획을 먼저 받고 승인"하는 구조를 만들 때 씁니다.

```
(Plan 모드에서)
"user 테이블에 last_login 칼럼 추가하는 마이그레이션 플랜 짜 줘"

→ 클로드가 파일은 안 건드리고 계획만 답
→ "좋아, 실행해"라고 하면 Normal로 돌아와 실제 작업
```

`/plan` 커맨드로도 진입.

## 모드별 조합 전략

### 탐색형 세션 시작

1. Plan 모드로 시작
2. "이 코드베이스에서 인증 흐름이 어디 있어?" 조사
3. 계획 확정 → Normal로 전환 → 구현

### 고속 반복

1. Normal로 설계
2. 구현 단계에서 Auto-accept로 전환
3. 긴 리팩터링 중 자리 비워도 자동 진행
4. 끝나면 Normal로

### 위험한 작업

1. Auto-accept였으면 Normal로 되돌림
2. 작업 수행
3. 필요하면 Plan 모드로 영향 분석

## 파일·이미지·입력 소스

### 파일 참조

앞서 설명한 `@` 외에도:

- **드래그 앤 드롭** — 파일·폴더를 입력창에 드래그하면 경로가 자동 입력
- **스크린샷 붙여넣기** — `Cmd+Ctrl+Shift+4`로 캡처 후 `Ctrl+V`

### 이미지 입력

- **클립보드** — `Ctrl+V`로 붙여넣기 (macOS는 `Cmd+V`)
- **파일 경로** — `@screenshots/error.png`
- **드래그 앤 드롭** — PNG/JPG/GIF/WebP 지원

에러 스크린샷, 디자인 시안, 화이트보드 사진을 그대로 보여주면 설명 없이 상황 전달됩니다.

### 명령 출력 파이프

셸에서 클로드 코드로 파이프:

```bash
git diff | claude "이 diff 한국어로 요약해 줘"
cat error.log | claude "에러 원인 분석"
```

헤드리스 모드(`claude -p`)와 결합하면 CI·스크립트에 녹이기 쉽습니다.

## Vim 모드

터미널에 익숙한 사람을 위해 Vim 키바인딩 지원.

```
/vim
```

입력창에서 Esc로 Normal 모드, `i`로 Insert, `h/j/k/l` 이동 등 기본 Vim 조작. 다시 끄려면 한 번 더 `/vim`.

## 터미널 세팅

`/terminal-setup`으로 현재 터미널의 최적 설정 가이드를 받습니다. 권장:

- **iTerm2 / Warp / Ghostty** — 트루컬러와 이미지 미리보기 지원
- **Font** — 유니코드·이모지 렌더링 잘 되는 Nerd Font류 권장
- **Line height** — 1.2~1.4 사이가 읽기 편함

## 상태줄

기본 상태줄엔 현재 모드, 연결된 IDE, 토큰 사용량 등이 표시됩니다. 커스터마이즈하려면 settings.json의 `statusLine`.

```json
{
  "statusLine": {
    "type": "command",
    "command": "~/.claude/statusline.sh"
  }
}
```

지정한 스크립트가 JSON 입력을 받아 한 줄 문자열을 출력하면 그게 상태줄에 표시. 브랜치명, 이슈 번호, 세션 단가 같은 걸 실시간으로 볼 수 있습니다.

## 커스텀 키바인딩

`~/.claude/keybindings.json`에 추가.

```json
{
  "bindings": {
    "ctrl+s": "skill:ko-review",
    "cmd+shift+p": "menu:all-commands"
  }
}
```

설정 후 `/keybindings-help`로 확인. 자주 쓰는 스킬이나 커맨드를 한 키로 올려 두면 체감 속도가 크게 달라집니다.

## 자주 하는 실수

### Auto-accept 켠 채 위험 명령

"Shift+Tab 두 번 눌러서 Plan 가려고 했는데 한 번만 눌러 Auto-accept였다"는 사고가 흔합니다. 상태줄 항상 확인.

### `!` Bash 모드 남용

`!` 모드는 결과를 클로드가 자동 해석하지 않습니다. "이 결과 설명해 줘"를 이어서 물어야 함. 반대로 클로드가 해석까지 해야 한다면 일반 대화로 "터미널에서 `ls` 결과 알려 줘" 식으로 부탁.

### `@` 남발

대용량 폴더를 `@` 한 번으로 통째 넣으면 토큰이 폭발합니다. 필요한 파일만 구체적으로 지정하거나, Explore 에이전트에게 "이 폴더에서 관련 파일 찾아와" 위임.

### Esc 한 번 vs 두 번

- 한 번: 진행 중인 작업 중단 (대화는 유지)
- 두 번: 체크포인트로 되돌리기 (직전 메시지 편집)

헷갈려서 작업 중단하려다가 대화를 되감는 실수 주의.

### 입력 히스토리 오염

`Ctrl+R`로 역검색하는 히스토리는 **세션 전체**에 걸쳐 쌓입니다. 비밀번호를 입력창에 쳤다면 즉시 `Ctrl+L`·세션 재시작으로 날리세요.

## 요약

입력 문법 네 개(`/@#!`)와 모드 세 개(`Shift+Tab`)만 몸에 익혀도 체감 생산성이 두 배입니다. 오늘 한 번 해 볼 일:

- `Esc` 두 번 체크포인트를 의식적으로 세 번 사용
- Plan 모드로 한 번 시작해 계획 받고 Normal로 전환
- 자주 쓰는 스크립트 하나를 `!` 모드 대신 스킬로 승격

일주일 쓰면 손가락이 기억합니다.
