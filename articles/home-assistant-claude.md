# 홈어시스턴트 × 클로드로 집에 음성 비서 들이기

> 시리보다 똑똑하고, 알렉사보다 내 말 잘 알아듣고, 구글홈보다 프라이버시 걱정이 덜한 음성 비서를 집에 들이는 법. 핵심은 **홈어시스턴트의 오픈 표준**과 **클로드의 추론**입니다.

## 들어가며

스마트 스피커 한 번쯤 사 본 분들은 같은 경험을 합니다. 처음 몇 주는 "와, 되네?" 하다가, 반년 지나면 **타이머랑 음악만 틀게 되죠.** 자연어 이해가 경직되어 있고, 내 집 기기 상태를 맥락으로 읽지 못하고, 사생활 데이터가 어디로 가는지도 찜찜합니다.

[홈어시스턴트(Home Assistant)](https://www.home-assistant.io/)는 이 문제를 "**오픈소스 허브**"로 풀어 온 프로젝트입니다. 필립스 휴·샤오미·LG ThinQ·삼성 SmartThings·애플 홈 — 가리지 않고 품어 하나의 대시보드에 모읍니다. 그런데 2024~2025년 사이에 진짜 재미있는 게 붙었어요.

1. **LLM Conversation Agent** — 음성 비서의 두뇌를 GPT·Claude·로컬 모델로 교체 가능
2. **MCP Server 통합** — 홈어시스턴트 자체가 MCP 서버가 되어, Claude Desktop/Code에서 집을 직접 제어
3. **풀 로컬 보이스 파이프라인** — STT·의도 인식·TTS를 전부 내 집 안에서

이 세 축을 엮으면 "시리 + 알렉사 + 우리 엄마의 직관"을 합친 개인용 비서가 됩니다. 이 글에서는 그 세팅을 단계별로 정리합니다.

## 1. 준비물

하드웨어는 가벼워도 됩니다.

- **홈어시스턴트 설치본** — 전용 [Home Assistant Green/Yellow](https://www.home-assistant.io/green/)(플러그 앤 플레이) 또는 라즈베리파이 4/5, 중고 NUC, Proxmox 가상머신 아무거나
- **Anthropic API 키** — [console.anthropic.com](https://console.anthropic.com/)에서 발급. 무료 크레딧부터 시작해도 됨
- **스마트 기기 몇 개** — 조명·스위치·센서 아무거나. 없으면 일단 모바일 앱으로 "가짜 기기"만 만들어 실험 가능
- (선택) 마이크 내장 스피커 — [Voice PE](https://www.home-assistant.io/voice-pe/), M5 ATOM Echo, 또는 Android 폰 앱

홈어시스턴트 설치 자체는 공식 가이드가 워낙 잘 되어 있어 이 글에서는 생략합니다. 아래는 **이미 HA가 떠 있다**는 전제에서 시작합니다.

## 2. 클로드를 "음성 비서의 두뇌"로 앉히기 (경로 A)

가장 빠른 경로는 홈어시스턴트에 **공식 Anthropic 통합**을 붙여 Conversation Agent로 쓰는 것입니다.

### 설치

1. HA 설정 → **Integrations** → **Add Integration** → "Anthropic" 검색
2. API 키 입력
3. 모델 선택 (Claude Sonnet/Opus/Haiku)
4. **Control Home Assistant** 체크박스 활성화 → Claude가 HA 엔티티를 읽고 조작할 수 있게 됨

### 어시스트 파이프라인에 연결

**Settings → Voice Assistants → Assist**에서 새 파이프라인을 만들고 Conversation Agent로 방금 만든 Anthropic을 선택합니다. 이제 HA 앱에서 마이크를 탭하거나, Voice PE에 대고 말하거나, 대시보드의 Assist 버튼을 누르면 Claude가 대답합니다.

### 바로 시험해 볼 요청들

```text
"거실 불 따뜻한 색으로 30% 밝기로 바꿔줘"
"오늘 집에 누가 언제 들어왔어?"
"지금 집안에 켜져 있는 전기 기기들 하나씩 정리해서 보여줘"
"자기 전 루틴 만들어 줘 — 조명 점점 어둡게, 문 잠그고,
 창문 열려 있으면 알림"
```

Claude는 HA의 엔티티 상태를 **문맥으로 읽어** 답합니다. "지금 집안 온도"는 온도 센서를 조회하고, "문 열려 있으면"은 door sensor를 체크합니다. 시리가 못 하는 일이 여기서부터 시작됩니다.

## 3. 집 밖에서도, 다른 앱에서도 제어하기 (경로 B)

경로 A는 **집 안의 비서**를 만들어 줍니다. 경로 B는 반대로 **클로드가 있는 곳에서 집을 제어**하는 구조예요.

### MCP Server 통합 활성화

HA 2025.2부터 [Model Context Protocol Server](https://www.home-assistant.io/integrations/mcp_server/) 통합이 정식 추가됐습니다.

1. Integrations에서 "MCP Server" 추가
2. Long-Lived Access Token 발급 (프로필 → 하단 "Long-Lived Access Tokens")
3. HA의 외부 접근 URL 확인 (예: `https://my.homeassistant.example.com`)

### Claude Desktop/Code에 연결

**Claude Code:**

```bash
claude mcp add home-assistant -- npx -y @home-assistant/mcp-client \
  --url https://my.homeassistant.example.com \
  --token <YOUR_LONG_LIVED_TOKEN>
```

**Claude Desktop** — 설정의 MCP 섹션에서 원격 MCP 서버로 HA URL + 토큰을 붙여 넣습니다.

이제 Claude 세션에서 자연어로 집을 조작할 수 있습니다. "회의 시작하려는데 주변 노이즈 될 만한 거 다 꺼줘" 같은 요청을 사무실 책상에서 던지면, Claude가 집의 로봇청소기·가습기·세탁기를 멈추게 합니다.

### 언제 경로 A, 언제 경로 B?

| 상황 | 추천 경로 |
|------|-----------|
| 집에서 말로 불러서 쓰고 싶다 | A (HA 안에 Claude 내장) |
| 이미 Claude 쓰고 있고 집도 제어하고 싶다 | B (MCP 서버) |
| 원격지·여행 중 제어 | B |
| 루틴/자동화 설계 | A가 편함 |
| 복잡한 작업 묶기 (캘린더·이메일·집) | B |

둘 다 설치해도 충돌 안 납니다. 대부분의 사용자는 **둘 다** 켜 둡니다.

## 4. 완전 로컬 보이스 파이프라인 (경로 C, 심화)

"API 호출이 클라우드로 나가는 게 싫다"면 보이스 파이프라인의 각 단계를 로컬로 구성할 수 있습니다.

**홈어시스턴트 Voice Chapter 10** (2025-06)부터 음성 인식·의도 인식·음성 합성을 각각 다른 기기에서 돌릴 수 있는 **Wyoming Protocol**이 표준입니다.

| 단계 | 로컬 옵션 | 클라우드 옵션 |
|------|-----------|---------------|
| 마이크 | Voice PE / ATOM Echo / 폰 | — |
| 음성 인식 (STT) | faster-whisper (Wyoming) | Google Cloud STT |
| 의도 이해 | HA 내장 → LLM 폴백 | — |
| 대화 엔진 | Ollama + Llama 3 | Anthropic Claude |
| 음성 합성 (TTS) | Piper (Wyoming) | ElevenLabs |

가장 현실적인 **하이브리드**:

- STT: 로컬 faster-whisper
- 일반 명령("거실 불 꺼줘"): HA 내장 의도 엔진이 즉시 처리 (지연 0.3초)
- 복잡 질의("오늘 저녁 뭐 먹지, 냉장고에 뭐 있어?"): Claude로 폴백
- TTS: 로컬 Piper

이 구성이면 전기·등·난방 같은 **일상 명령은 100% 오프라인**에서 돌고, 고민이 필요한 순간만 Claude가 붙습니다. 응답 속도와 프라이버시를 동시에 챙기는 균형점입니다.

## 5. 실전 시나리오 — 집에 없던 기능들

경로 A+B를 켠 상태에서 일상에 녹는 루틴 몇 가지.

### "집에 왔어"

현관 센서가 열리면 HA가 위치·시간을 맥락으로 넘기고, Claude가 상황에 맞는 루틴을 선택합니다.

- **평일 저녁** → 조명 웜톤, 커튼 닫기, "오늘 택배 2건 있었어요, 현관에 둬뒀을 거예요"
- **늦은 밤** → 조용한 모드, 거실 대신 침실 쪽만 환영등
- **주말 오후** → 오디오 켜기, "날씨 괜찮으니 베란다 열까요?"

### "잘 자"

```text
"잘 자" → Claude 처리
  1. 켜진 조명·TV 전부 확인
  2. 문·창문 센서 스캔
  3. 침실 조명만 남기고 서서히 페이드아웃
  4. "창문 두 개 열려 있어요. 닫을까요?"
  5. 내일 알람·일정 요약 낭독
  6. 백색소음 시작
```

이걸 시리·알렉사로 하려면 10개 이상의 "루틴"을 수동으로 엮어야 합니다. Claude는 **상황을 보고 즉석에서 구성**합니다.

### "회의 시작"

캘린더에 회의 일정이 있고, Claude가 이를 MCP로 읽습니다.

```text
"곧 회의야" → Claude
  - 에어컨 소음 줄이기 (약풍 모드)
  - 로봇청소기 일시정지
  - 거실 스마트 스피커 음소거
  - 현관 초인종 무음화 (택배 기사 혼선 방지)
  - 회의 참석자 이름을 일정에서 읽어 "오늘 OO님과 미팅이에요"
```

### "몸 안 좋아"

센서 데이터 + 캘린더 + 자연어 이해가 만나는 지점.

```text
"머리가 좀 아파" → Claude
  - 실내 CO₂ 센서 확인 → 1400ppm으로 높음
  - 창문 열기 요청
  - 조명 톤 내리기
  - 내일 회의 중 이동 없는 것만 남기고 리스케줄 제안
  - 타이머 15분 설정, 휴식 후 다시 상태 확인
```

## 6. 커스텀 인텐트와 LLM 폴백의 균형

모든 걸 LLM에 맡기면 **느리고 비쌉니다.** 반대로 전부 규칙 기반이면 **경직됩니다.** 권장하는 계층:

1. **커스텀 인텐트** — 자주 쓰는 패턴("거실 불 켜줘", "에어컨 25도")은 HA의 `custom_sentences`로 고정. 0.3초 처리
2. **HA 내장 의도** — 일반 기기 제어
3. **LLM 폴백** — 나머지 전부

HA의 Conversation 설정에서 `prefer_local_intents: true`를 두면 커스텀 인텐트가 먼저 시도되고, 매칭 실패 시에만 Claude로 넘어갑니다. 비용·지연이 확 줄어듭니다.

예시 `custom_sentences/ko/light.yaml`:

```yaml
language: "ko"
intents:
  HassTurnOn:
    data:
      - sentences:
          - "{area} (불|조명) (켜|켜줘|켜 줘)"
          - "{area} 조명 {brightness} 퍼센트로"
```

이런 패턴 50개만 정의해도 "일상 명령의 80%"가 오프라인으로 처리됩니다.

## 7. 프라이버시·비용·지연 — 현실 체크

### 프라이버시

- 경로 A/B는 API 요청이 Anthropic으로 나갑니다. 위치·기기 상태가 Claude에게 전달됨
- 민감 데이터(헬스·카메라 이미지)는 LLM에 넘기지 않도록 **Expose 설정**에서 엔티티별로 차단
- 완전 로컬이 필요하면 경로 C + Ollama 조합

### 비용

- Claude Haiku 기준으로 일반 가정 사용(하루 20~30 요청)이면 월 1~3달러 수준
- Sonnet/Opus로 올리면 5~15달러
- 음성 STT/TTS를 클라우드 쓰면 추가 과금. 로컬로 돌리면 전기세만

### 지연

- 경로 A 전체 왕복: 보통 1.5~3초 (STT → API → TTS)
- 커스텀 인텐트로 폴백 안 하면: 0.3~0.8초
- 로컬 LLM(Ollama + 24GB VRAM): 2~4초, 단 품질은 Claude보다 낮음

체감상 **"불 켜줘"는 0.5초 안에 되어야** 자연스럽습니다. 그 선을 지키려면 커스텀 인텐트 투자가 필수입니다.

## 8. 함정들

### 엔티티 노출 관리

Claude에게 **모든 엔티티를 노출하면** 프롬프트가 비대해져 비용·지연이 폭발합니다. HA의 **Expose to Assistants** 설정에서 자주 쓰는 조명·센서·스위치만 노출하세요. 카메라 스트림·로그 엔티티는 끄기.

### 자동화 vs 대화

복잡한 자동화는 **자동화 엔진(HA Automation/Scripts)**으로 만들고, Claude에게는 그 자동화를 **호출만** 시키는 편이 안정적입니다. LLM이 매번 처음부터 로직을 조립하면 결정론이 떨어집니다.

### API 키 보관

HA는 secrets.yaml에 API 키를 보관합니다. HA 설정 백업을 클라우드 동기화 한다면, 백업에 키가 포함되지 않도록 **exclude 규칙**을 확인하세요.

### 오동작 복구

LLM이 잘못된 기기를 조작한 경우를 대비해 **"취소"·"원복"** 음성 명령을 커스텀 인텐트로 만들어 두세요. HA의 scene 기능으로 "직전 상태"를 저장해 두면 즉시 되돌릴 수 있습니다.

## 9. 더 해 볼 만한 것들

세팅이 익었다면 확장할 수 있는 방향:

- **다중 거주자** — 목소리 인식으로 사용자 구분, 개인별 일정 연동
- **에너지 절약 AI** — 전력 패턴 학습해 "지금 세탁 돌리면 전기료 절감" 제안
- **반려동물 돌봄** — 카메라 + 피더 + Claude로 외출 중 상태 리포트
- **홈 시큐리티** — 이상 패턴 감지 시 요약 리포트, 오탐 필터링
- **어른 돌봄** — 노부모 댁 HA + Claude로 원격 체크 (낙상 알림 톤 조절)

## 마치며

클라우드 음성 비서들이 10년 간 해 온 것을, 홈어시스턴트와 Claude를 엮으면 **주말 하루** 만에 넘어섭니다. 비결은 "내 집의 오픈 표준 허브"와 "맥락을 이해하는 LLM"의 조합입니다. 시리·알렉사가 고정된 명령어 몇 백 개를 외우게 한 것이었다면, 이 조합은 **내가 말하는 방식을 비서가 맞춰 오는** 쪽입니다.

오늘 저녁, HA에 Anthropic 통합 하나 붙이는 것부터 시작해 보세요. 한 달 뒤엔 "이거 없던 시절에 어떻게 살았지" 싶어질 겁니다.

```bash
# 가장 빠른 시작
1. HA → Integrations → Anthropic 추가 (API 키만)
2. Voice Assistants → 파이프라인 생성 → Conversation Agent: Anthropic
3. 모바일 앱에서 마이크 탭 → "거실 불 좀 따뜻하게"
```
