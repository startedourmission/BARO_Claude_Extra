## 모델 선택 기준

| 작업 | 권장 |
|------|------|
| 새 기능 설계 | Opus |
| 일반 구현 | Sonnet |
| 파일 리네임·커밋 메시지 | Haiku |
| 복잡 디버깅 | Opus |

## 편집 원칙

- 파일 전체 재작성은 마지막 수단. 변경 부분만 Edit로 교체
- 같은 세션에서 같은 파일을 두 번 Read하지 않음
- 긴 산출물은 파일로 저장, 대화에는 완료 확인 한 줄만

## 컨텍스트 관리

- 주제 전환 시 /clear
- 긴 세션 중간에 `/compact 코드 샘플과 API 시그니처 보존, 탐색 대화는 버려`
- CLAUDE.md는 150줄 이내 유지

## 서브에이전트 위임

탐색·조사 작업은 Explore 에이전트에 위임. 결과 요약만 메인으로 받음.

## 스크립트 우선

결정론적 연산(파싱, 집계, 포맷)은 scripts/로 이관.
LLM은 판단과 요약만 담당.
